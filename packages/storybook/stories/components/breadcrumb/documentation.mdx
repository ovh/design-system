## Anatomy

![Component anatomy](components/breadcrumb/anatomy.png "Component anatomy")
1. **Page link** (as Home [**Icon**](?path=/docs/ods-components-icon--documentation))
2. **Page link**: Directs users to the parent-level page.
3. **Separator**: Element between each page.
4. **Active page** (Non-clickable): Indicate the current page.

## Usage

It has several usage :
* Displaying sub-pages of a site structure
* Show a step progress of a process
* Simplify site structure navigation in a quicker way

### Dos & Don'ts

<Markdown>
  { `
| ✅ Do                                                                                                                                                                           |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| - The first **Breadcrumb** element has to be a [**Link**](?path=/docs/ods-components-link--documentation) to its preferential home page (with an [**Icon**](?path=/docs/ods-components-icon--documentation), or with "Home" [**Text**](?path=/docs/ods-components-text--documentation))
| - Inherit page structure logic inside the **Breadcrumb** construction
` }
</Markdown>

<Markdown>
  { `
| ❌ Don't                                                                                                                                                                          |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Use **Breadcrumb** for main page navigation                                                                                                                                    |
| - Use nested **Breadcrumbs**                                                                                                                                                     |
| - Add external [**Link**](?path=/docs/ods-components-link--documentation) in the component                                                                               |
| - Put more than 4 imbrications inside a **Breadcrumb**, as the structure will be too complex                                                                                     |
` }
</Markdown>

## Placement

A **Breadcrumb** is used at the top of a web page, preferably start-aligned.

Its width is automatic, relative to its content and is not adjustable.

## Behavior

When the **Breadcrumb** has more than 4 [**Link**](?path=/docs/ods-components-link--documentation) visible, an ellipsis is displayed as a replacement for the middle [**Link**](?path=/docs/ods-components-link--documentation).

A click on the ellipsis will expanded all previously hidden [**Link**](?path=/docs/ods-components-link--documentation) inline, the collapsed state can't be redone afterwards.

The **Breadcrumb** [**Link**](?path=/docs/ods-components-link--documentation) are kept inline, even on mobile viewports.

Clicking or pressing `Enter` or `Space` key on **Breadcrumb** item triggers an event.

Same actions on **Breadcrumb** collapsed item will trigger an event.
