## Usage

A **Card** can be used to:

* display a wide variety of content such as a [**Text**](?path=/docs/ods-components-content-tag--documentation), a [**Link**](?path=/docs/ods-components-actions-link--documentation) or other elements like a [**Button**](?path=/docs/ods-components-actions-button--documentation)
* emphasize important content
* visually group related content together
* guide the user toward actions
* provide the option to navigate to more detailed content

Several elements can be used in a **Card**:

* A title
* A description
* A [**Button**](?path=/docs/ods-components-actions-button--documentation)
* A [**Medium**](?path=/docs/ods-components-content-medium--documentation)
* Extra information

All these elements are not mandatory in every **Card**.

### Dos & Don'ts

| - Use content in a **Card**, such as [**Text**](?path=/docs/ods-components-content-tag--documentation), [**Medium**](?path=/docs/ods-components-content-medium--documentation) and [**Button**](?path=/docs/ods-components-actions-button--documentation), that are contextually related |

` }
</Markdown>
<Markdown>
  { `
| ❌ Don't                                                                                                                                                                          |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Overuse **Cards** since it may confuse users                                                                                                                             |
| - Add too much content in a **Card**                                                                                                                                       |
| - Group **Cards** that are unlike and use different elements                                                                                                               |
` }
</Markdown>

## Placement

**Cards** have an automatic width, but they can widen to match the container.

**Cards** that are strongly related can be grouped in a layout group. This group can flow horizontally left to right and/or vertically top to bottom.
