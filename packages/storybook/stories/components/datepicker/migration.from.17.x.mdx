# Datepicker - migrate from v17 to v18
----

## Attributes changes

`ariaLabel` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

You can use the new `ariaLabel` attribute to apply the aria label value on the input.

`ariaLabelledby` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

You can use the new `araiLabelledby` attribute to apply the aria labelled-by value on the input.

`clearable` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isClearable` attribute to obtain the same behavior.

`color ` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Form components does not provide color variant anymore.

`disabled` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`error` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `hasError` attribute to obtain the same behavior.

`inline ` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Datepicker is now `inline-block` by default.
You can add some style directly on the component if you want to override that.

`isLoading` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

You can use the new `isLoading` attribute to apply the loading state on the input.

`isReadonly` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

You can use the new `isReadonly` attribute to apply the readonly state on the input.

`isRequired` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

You can use the new `isRequired` attribute to apply the required state on the input.

`maxDate` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `max` attribute to obtain the same behavior.

`minDate` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `min` attribute to obtain the same behavior.

`showSiblingsMonthDays` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

As it wasn't necessary anymore.

If you were actually using this attribute, feel free to reach out to us, so that we can get a better
understanding of your actual need and offer the best solution.


## Migration examples

Clearable date picker:
```html
<osds-datepicker clearable>
</osds-datepicker>

<!-- is now -->

<ods-datepicker is-clearable>
</ods-datepicker>
```

Disabled date picker:
```html
<osds-datepicker disabled>
</osds-datepicker>

<!-- is now -->

<ods-datepicker is-disabled>
</ods-datepicker>
```

Errored date picker:
```html
<osds-datepicker error>
</osds-datepicker>

<!-- is now -->

<ods-datepicker has-error>
</ods-datepicker>
```
