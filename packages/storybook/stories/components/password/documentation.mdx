<Heading label="Anatomy" level={ 2 } />

![Component anatomy](components/password/anatomy.png "Component anatomy")

1. **Placeholder**: it provides hints or example of what user should type
2. **Container**: it is a container where the user can type in
3. **Clear action**: a `close` icon <StorybookLink kind={ ODS_COMPONENTS_TITLE.button } story={ STORY.documentation }>Button</StorybookLink> so that user can clear the <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } story={ STORY.documentation }>Input</StorybookLink> content (optional)
4. **Show/hide action**: an `eye` icon <StorybookLink kind={ ODS_COMPONENTS_TITLE.button } story={ STORY.documentation }>Button</StorybookLink> to toggle between masked or visible <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } story={ STORY.documentation }>Input</StorybookLink> content

<Heading label="Usage" level={ 2 } />

A **Password** is mainly used to let users enter a password or secret information.

Entered password can be masked or visible as plain text when users need to check what they have entered.

It can also be used for displaying tokens.

<Heading label="Dos & Don'ts" level={ 3 } />

<BestPractices
  donts={[
    <span>- Use a Password when you need the text users enter to be only visible. You should use an <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } story={ STORY.documentation }>Input</StorybookLink> instead</span>,
  ]}
  dos={[
    '- Use a Password when user has to enter sensitive data in the field',
    '- Use read-only Password for tokens with copyable action enabled',
  ]}
/>

<Heading label="Placement" level={ 2 } />

By default, the **Password** content is left-aligned in its container.

<Heading label="Behavior" level={ 2 } />

By default, the **Password** content is masked.

**Password** can be used as read-only (especially for tokens).

The password masking toggled using show/hide action is permanent. Users have to click again to show/hide the <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } story={ STORY.documentation }>Input</StorybookLink> content.

It inherits events from <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } story={ STORY.documentation }>Input</StorybookLink> component.
