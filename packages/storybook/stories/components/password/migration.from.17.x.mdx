# Password - migrate from v17 to v18
----

## Attributes changes

`color ` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Form components do not provide color variants anymore.
You can use custom CSS if you need to customize the component.

`contrasted` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Form components do not provide contrasted variants anymore.
You can use custom CSS if you need to customize the component.

`clearable` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isClearable` attribute to obtain the same behavior.

`disabled` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`forbidden-values` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

You can replace it with a regex on the `pattern` attributes.

`label` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

To add a label on a password component, wrap it within `form-field` component.

`loading` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isLoading` attribute to obtain the same behavior.

`masked` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isMasked` attribute to obtain the same behavior.

`error` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `hasError` attribute to obtain the same behavior.

`readonly` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isReadonly` attribute to obtain the same behavior.

`required` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isRequired` attribute to obtain the same behavior.

`inline` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Password is now `inline-block` by default.
You can add some style directly on the component if you want to override that.

`size ` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Form components do not provide size variants anymore.
You can use custom CSS if you need to customize the component.

## Migration examples

Clearable password:
```html
<osds-password clearable>
</osds-password>

<!-- is now -->

<ods-password is-clearable>
</ods-password>
```

Disabled password:
```html
<osds-password disabled>
</osds-password>

<!-- is now -->

<ods-password is-disabled>
</ods-password>
```

Error password:
```html
<osds-password error>
</osds-password>

<!-- is now -->

<ods-password has-error>
</ods-password>
```

<ods-password pattern="[^12]+">
