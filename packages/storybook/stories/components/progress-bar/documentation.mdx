## Anatomy

![Component anatomy](components/progress-bar/anatomy.png "Component anatomy")

1. **Track**: displays the visual range of progression, from a starting to an ending position
2. **Filled track**: represents the visual progression value within the **Progress Bar**.

## Usage

**Progress Bar** component is only used in process progression, such as submitting, uploading or saving items.

### Dos & Don'ts
<Markdown>
  { `
| ✅ Do                                                                                                                                                                                                                                                                                                                                    |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------                                                                                                                                                          |
| - Use **Progress Bar** for quantifiable loading; if not, please use [**Spinner**](?path=/docs/ods-components-user-feedback-spinner--documentation) for complex content or [**Skeleton**](?path=/docs/ods-components-content-skeleton--documentation) for [**Text**](?path=/docs/ods-components-content-tag--documentation)-only content |
` }
</Markdown>
<Markdown>
  { `
| ❌ Don't                                                                                                                                                                          |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Use **Progress Bar** for data visualization                                                                                                                                    |
| - Use all three starting value, ending value and progress value labels at the same time                                                                                          |
` }
</Markdown>

## Placement

**Progress Bar** is usually centered in its container, and can be stretched to match the container width if necessary.

## Behavior

**Progress Bar** is filled from the minimum to the maximum value, depending on the progress described.
