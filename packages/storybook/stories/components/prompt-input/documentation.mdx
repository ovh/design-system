<Heading label="Usage" level={ 2 } />

Use the **Prompt Input** component whenever users need to compose and submit a message or query in a single interaction.

Common use cases include:

- AI assistant interfaces
- Customer support or live chat widgets

<Heading label="Placement" level={ 2 } />

The **Prompt Input** is typically anchored at the bottom of a conversational view, spanning the full width of the content area.

It should remain visible and accessible at all times during an active conversation, without being obscured by the message list above it.

<Heading label="Behavior" level={ 2 } />

The **Prompt Input** supports several states:

- **Default** - the user can type freely in the textarea, which grows vertically as content is added.
- **Disabled** - the textarea, file upload button, and send button are all non-interactive.
- **Read-only** - the textarea displays its current value but does not accept new input.
- **Loading** - the textarea and both buttons are disabled, and the send button displays a spinner.

The send button is automatically disabled whenever the textarea is empty, disabled, or in a loading state — no manual control of the button is required.

<Heading label="Keyboard Interaction" level={ 2 } />

Pressing <Kbd>Enter</Kbd> triggers `onInputSubmit` (equivalent to clicking the send button).

Pressing <Kbd>Shift</Kbd> + <Kbd>Enter</Kbd> inserts a new line without submitting.

Pressing <Kbd>Tab</Kbd> moves focus to the next interactive element (send button, file upload button).

Pressing <Kbd>Shift</Kbd> + <Kbd>Tab</Kbd> moves focus to the previous focusable element.

<Heading label="Accessibility" level={ 2 } />

<Heading label="Textarea labelling" level={ 3 } />

Because the **Prompt Input** does not render a visible label by default, you must supply an accessible name for the textarea.

Use the `aria-label` prop on `PromptInputTextControl` to give screen readers a meaningful description of the field.

<Canvas of={ PromptInputStories.AccessibilityAriaLabel } sourceState="shown" />

<Heading label="Button labelling" level={ 3 } />

- The file upload button and the send button don't hold accessible text by default, therefore they must carry an `aria-label` (e.g. `"Attach file"`, `"Send request"`, …) to be announced correctly by assistive technologies.
