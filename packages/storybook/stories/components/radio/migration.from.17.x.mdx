# Radio - migrate from v17 to v18
----

## Attributes changes Radio

`checked` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isChecked` attribute to obtain the same behavior.

`checking` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`disabled` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`label` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Radio can now use HTML Label. Like in the example.

`afterSave` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

We don't think this kind of callback was necessary anymore.

`beforeSave` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

We don't think this kind of callback was necessary anymore.

`save` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

We don't think this kind of callback was necessary anymore.

## Migration examples

Disabled radio:
```html
<osds-radio disabled></osds-radio>

<!-- is now -->

<ods-radio is-disabled></ods-radio>
```

Label radio:
```html
<osds-radio label="Radio label"></osds-radio>

<!-- is now -->

<ods-radio input-id="migration-label" name="name" value="migration"></ods-radio>
<label for="migration-label">Radio label</label>
```
