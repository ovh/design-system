<Heading label="Anatomy" level={ 2 } />

![Component anatomy](components/range/anatomy.png "Component anatomy")

1. **Track**: the range of values the user can select and adjust
2. **Filled track**: the user's selected value within the **Range**. It is filled from the minimum value to the selected value
3. **Thumb**: a positional indicator. It can be moved along the track to indicate a position on this track
4. **Additional thumb**: a second positional indicator to get a dual **Range**. It allows users to select a portion within the range _(optional)_
5. **Minimum value label**: beginning of a **Range** as its minimum value
6. **Maximum value label**: end of a **Range** as its maximum value

<Heading label="Usage" level={ 2 } />

A **Range** is used to select a numeric value within a given range with defined minimum and maximum values.

It is easily adjustable in a visually pleasing interface and, after interacting with it, changes are reflected immediately.

A dual **Range** allows the user to select a numeric range value that is no less than a given value, and no more than another given value.

<Heading label="Dos & Don'ts" level={ 3 } />

<BestPractices
  donts={[
    '- Overuse Ranges on one page',
    '- Use a Range when the numeric value range is infinite',
    '- Use a Range in too small spaces on a page: it is necessary to have enough space in the UI',
    '- Use a continuous Range when the defined range of values is too large: use a discrete Range',
  ]}
  dos={[
    '- Use a Range when the user needs a way to select a value within a range',
    <span>- Include a label on a Range, such as the one provided by <StorybookLink kind={ ODS_COMPONENTS_TITLE.formField } label="Form Field" story={ STORY.documentation } /> component, to explain the purpose of the Range</span>,
    '- Add a default numeric value',
  ]}
/>

<Heading label="Placement" level={ 2 } />

A **Range** can be used in a page as long as there is a need to allow users to select a numeric value within a given range, for a quantity or a volume for example.

It has a width by default, but it can widen to match its container.

The **Range** can also be used inside components.

It also can be combined with other elements such as an <StorybookLink kind={ ODS_COMPONENTS_TITLE.input } label="Input" story={ STORY.documentation } /> for a better control of the value selection.

<Heading label="Behavior" level={ 2 } />

A **Range** can be focused and hovered. It also can be disabled.

The user can select a numeric value by clicking and dragging the thumb along the track.

An event is triggered when the **Range** value(s) changes.
