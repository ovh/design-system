# Range - migrate from v17 to v18
----

## Attributes changes

`color` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`disabled` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`error` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `hasError` attribute to obtain the same behavior.

`errorStateControl` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Was not relevant for few versions already.

`formControl` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Was not relevant for few versions already.

`forbidden-values` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

You can replace it with a regex on the `pattern` attributes.

`inline` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Range is now `inline-block` by default.
You can add some style directly on the component if you want to override that.

## Migration examples

Disabled range:
```html
<osds-range disabled>
</osds-range>

<!-- is now -->

<ods-range is-disabled>
</ods-range>
```

Error range:
```html
<osds-range error>
</osds-range>

<!-- is now -->

<ods-range has-error>
</ods-range>
```

Inline range:
```html
<osds-range inline>
</osds-range>

<!-- is now -->

<ods-range>
</ods-range>
```
