# Textarea - migrate from v17 to v18
----

## Attributes changes

`color ` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Form components does not provide color variant anymore.
You can use custom css if you need to customize the component.

`contrasted ` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Form components does not provide color variant anymore.
You can use custom css if you need to customize the component.

`disabled` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`error` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `hasError` attribute to obtain the same behavior.

`errorStateControl` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Was not relevant for few versions already.

`formControl` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Was not relevant for few versions already.

`inline` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Textarea is now `inline-block` by default.
You can add some style directly on the component if you want to override that.

`readonly` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isReadonly` attribute to obtain the same behavior.

`required` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isRequired` attribute to obtain the same behavior.

`resizable` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isResizable` attribute to obtain the same behavior.

`size ` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Form components does not provide size variant anymore.
You can use custom css if you need to customize the component.

`spellcheck` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `hasSpellcheck` attribute to obtain the same behavior.

`textAreaId ` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This was used to manage a default aria value which is not set by default.

```html

</osds-textarea>
```
