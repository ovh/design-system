# Toggle - migrate from v17 to v18
----

## Attributes changes

`checked` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `value` attribute to obtain the same behavior.

`checking` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`contrasted` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`color` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`disabled` <ods-badge size="sm" icon="circle-info" label="Updated"></ods-badge>

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`interactive` <ods-badge size="sm" color="critical" icon="circle-xmark" label="Removed"></ods-badge>

Has been removed.

Toggle is now always interactive.

`isRequired` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

Defines if the toggle is required in a form.

`name` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute.

You can use the new 'name' attribute to identify your OdsToggle within a form.

`withLabel` <ods-badge size="sm" color="success" icon="star" label="New"></ods-badge>

New attribute (optional).

Will add the visual labels `ON/OFF`.

## Migration examples

Checked toggle:
```html
<osds-toggle checked>
</osds-toggle>

<!-- is now -->

<ods-toggle value="true">
</ods-toggle>
```

Disabled toggle:
```html
<osds-toggle disabled>
</osds-toggle>

<!-- is now -->

<ods-toggle is-disabled>
</ods-toggle>
```
