# Toggle - migrate from v17 to v18
----

## Attributes changes

`checked` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `value` attribute to obtain the same behavior.

`checking` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`contrasted` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`color` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

This design feature has been removed in order to match the new styling of the component.

`disabled` <img src="https://img.shields.io/badge/updated-00FFFF" />

Has been updated.

You can use the new `isDisabled` attribute to obtain the same behavior.

`interactive` <img src="https://img.shields.io/badge/removed-FF0000" />

Has been removed.

Toggle is now always interactive.

`isRequired` <img src="https://img.shields.io/badge/new-008000" />

New attribute (optional).

Defines if the toggle is required in a form.

`name` <img src="https://img.shields.io/badge/new-008000" />

New attribute.

You can use the new 'name' attribute to identify your OdsToggle within a form.

`withLabel` <img src="https://img.shields.io/badge/new-008000" />

New attribute (optional).

Will add the visual labels `ON/OFF`.

## Migration examples

Checked toggle:
```html
<osds-toggle checked>
</osds-toggle>

<!-- is now -->

<ods-toggle value="true">
</ods-toggle>
```

Disabled toggle:
```html
<osds-toggle disabled>
</osds-toggle>

<!-- is now -->

<ods-toggle is-disabled>
</ods-toggle>
```
