## Usage

Use the **Tooltip** component to provide additional information or context for UI elements.

**Tooltips** should be brief and helpful, enhancing user understanding without overwhelming them.

They are suitable for [**Button**](?path=/docs/ods-components-button--documentation), [**Icon**](?path=/docs/ods-components-icon--documentation), [**Form Field**](?path=/docs/ods-components-form-elements-form-field--documentation), [**Link**](?path=/docs/ods-components-link--documentation), and any other elements where additional context would be beneficial.

### Dos & Don'ts

<Markdown className="documentation-do-table">

| - Use a **Tooltip** to provide useful, brief additional information.                                                                                                           |

| - Write **Tooltip**'s [**Text**](?path=/docs/ods-components-text--documentation) as full sentences with punctuation except there is not enough space.                   |

<Markdown className="documentation-dont-table">

| ❌ Don't                                                                                                                                                                                               |
|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------                      |
| - Use a **Tooltip** to communicate error messages.                                                                                                                                                    |
| - Write very important information within a **Tooltip** since it is only displayed on hover and focus so the user might miss its crucial content.                                                     |
| - Insert rich information such as image in a **Tooltip**.                                                                                                                                             |
| - Use interactive elements within a **Tooltip** such as [**Button**](?path=/docs/ods-components-button--documentation) or [**Link**](?path=/docs/ods-components-link--documentation). |
| - Repeat information within a **Tooltip** that is already visible in the UI.                                                                                                                          |
` }
</Markdown>

## Placement

**Tooltip** has automatic positioning feature. It can detect the edge of the browser so the container always stays visible on a page.

## Behavior

By default, a **Tooltip** is hidden to the user.

It triggers when the user hovers or focuses on the **Tooltip'**s trigger element such as an [**Icon**](?path=/docs/ods-components-icon--documentation).

As long as the user's mouse cursor or focus is active, the **Tooltip** will persist.

It will hide when the users stop hovering over or focusing on this element.

On a mobile device, a **Tooltip** is displayed upon tapping, and it will hide when the user taps an area outside the **Tooltip** container.

## Variation

- **With extra arrow**: providing a more pronounced visual connection between the tooltip and the element it describes, ensuring users clearly understand which element the tooltip is associated with.

## Accessibility

**Tooltip** component is a visual equivalent of ARIA labels, it will be read the same way by screen readers.
