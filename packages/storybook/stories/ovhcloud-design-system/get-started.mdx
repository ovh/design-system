<Meta title="OVHcloud Design System/Get Started" />

<Heading label="Get Started" level={ 1 } />

OVHcloud Design System (ODS) is a set of reusable React UI components crafted and tailored specifically for OVHcloud products. It embodies the OVHcloud brand expression and ensures a consistent user experience across all OVHcloud interfaces.

All OVHcloud Design System packages are available on <ExternalLink href="https://www.npmjs.com">NPM</ExternalLink>.

<Heading label="Installation" level={ 2 } />

Include **ODS** in your projects like any other npm/yarn/pnpm module:

<Source code="npm i --save-exact @ovhcloud/ods-react @ovhcloud/ods-themes"
        dark="true"
        language="bash" />
<Source code="yarn add --exact @ovhcloud/ods-react @ovhcloud/ods-themes"
        dark="true"
        language="bash" />
<Source code="pnpm add --save-exact @ovhcloud/ods-react @ovhcloud/ods-themes"
        dark="true"
        language="bash" />

<Heading label="Usage" level={ 2 } />

All components can be imported from the root path:

<Source code={`
import { Accordion, AccordionContent, AccordionItem, AccordionTrigger } from '@ovhcloud/ods-react';

const App = () => {
  return (
    <Accordion>
      <AccordionItem value="0">
        <AccordionTrigger>
          Hello World!
        </AccordionTrigger>
        <AccordionContent>
          Lorem ipsum dolor sit amet.
        </AccordionContent>
      </AccordionItem>
    </Accordion>
  );
};
`}
        dark="true"
        language="tsx" />

Each component is also available as compound component from its subpath:

<Source code={`
import { Accordion } from '@ovhcloud/ods-react/accordion';

const App = () => {
  return (
    <Accordion>
      <Accordion.Item value="0">
        <Accordion.Trigger>
          Hello World!
        </Accordion.Trigger>
        <Accordion.Content>
          Lorem ipsum dolor sit amet.
        </Accordion.Content>
      </Accordion.Item>
    </Accordion>
  );
};
`}
        dark="true"
        language="tsx" />

Components are compatible with React `v18.2+`, `v19+`.

<Heading label="Import components constants" level={ 2 } />

All (optional) enums, interfaces, events are available directly from the `ods-react` lib

<Source code={ `import { BUTTON_SIZE, type CheckboxProp } from '@ovhcloud/ods-react';` }
        dark="true"
        language="typescript" />

<Heading label="Import theme & fonts" level={ 2 } />

You need to import a theme to display all the components correctly.
It defines all the colors, typographies, etc...

via ES import:

<Source code={`
import '@ovhcloud/ods-themes/default/css';
import '@ovhcloud/ods-themes/default/fonts';
        `}
        dark="true"
        language="typescript" />

via Sass import:

<Source code={`
@import '@ovhcloud/ods-themes/default/css';
@import '@ovhcloud/ods-themes/default/fonts';
        `}
        dark="true"
        language="css" />

<Message dismissible={ false } style={{ width: '100%' }}>
  <MessageIcon name="circle-info" />

  <MessageBody>
    In case you encounter some issues while importing the font files, you can fallback to the base64 import using `import '@ovhcloud/ods-themes/default`.
    Please notice that this will increase the bundle size and prevent browser resources loading optimization.
  </MessageBody>
</Message>

<Heading label="Typescript" level={ 2 } />

ODS packages use modern `exports` fields in their `package.json`, which require the `bundler` module resolution strategy, released with <ExternalLink href="https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-0.html#--moduleresolution-bundler">Typescript 5.0</ExternalLink>.

<Source code={ `"moduleResolution": "bundler"` }
